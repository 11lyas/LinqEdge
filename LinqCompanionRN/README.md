# LINQ Companion — React Native (Expo)

A cross-platform (iOS + Android) patient lifestyle context app for active cardiac patients wearing Medtronic LINQ™ monitors. Built with React Native + Expo.

---

## What This App Does

LINQ monitors capture cardiac events — arrhythmias, elevated heart rates, AFib episodes — but have no context for *why* those events occurred. LINQ Companion lets patients log the surrounding lifestyle context:

| Logging | What it captures |
|---------|-----------------|
| **Workouts** | Activity type, duration, RPE, hydration, stress, symptoms during/after |
| **Symptoms** | Type (palpitations, dizziness…), severity 1–5, duration, context (during exercise, at rest…) |
| **Sleep** | Hours, quality, wake count, notes |

The app then auto-generates **insights** by correlating LINQ events with nearby logged data — "event 34 min after high-exertion run" or "event following poor sleep night" — and surfaces them in a **clinician-ready shareable report**.

> ⚠️ **Not a medical device.** No diagnostic capability. No direct connection to the LINQ monitor. All data stays on-device.

## Setup

### Prerequisites

- Node.js 18+
- [Expo CLI](https://docs.expo.dev/get-started/installation/): `npm install -g expo-cli`
- iOS: Xcode 15+ with iOS Simulator (or Expo Go on a physical device)
- Android: Android Studio with an emulator (or Expo Go)

### Install

```bash
cd LinqCompanionRN
npm install
```

### Run

```bash
# Start Expo dev server
npx expo start

# Specific platforms
npx expo start --ios       # iOS Simulator
npx expo start --android   # Android emulator
npx expo start --web       # Browser (limited native features)
```

Scan the QR code with **Expo Go** on your physical device, or press `i` for iOS Simulator / `a` for Android.

## State Management

All state lives in `PatientDataContext`, using `useReducer` + `AsyncStorage`:

```
AppState {
  userProfile         → saved to AsyncStorage on change
  onboardingComplete  → derived from userProfile
  linqEvents          → persisted
  workoutSessions     → persisted
  symptomEntries      → persisted
  sleepEntries        → persisted
}
```

**Derived state** (computed on every render, not persisted):
- `timelineEntries` — merged + sorted array across all data types
- `correlationInsights` — temporal pattern detection between events, workouts, symptoms, and sleep
- `generateReport(period)` — builds a `ReportData` struct for the chosen window

---

## Correlation Logic

Insights are auto-generated by `buildInsights()` in the context file. Pattern types:

| Pattern | Rule |
|---------|------|
| Activity Correlation | LINQ event within ±2 hours of a workout |
| Sleep Correlation | LINQ event the day after a Poor/Fair sleep night |
| No Symptom Event | Urgent LINQ event with no logged symptom within 4 hours |
| Palpitations Pattern | ≥2 instances of palpitations during exercise |
| Low Hydration Pattern | ≥2 workouts with hydration ≤2 and RPE ≥7 |

---

## Demo Mode

Onboarding lets the user choose **Demo Mode**, which loads 14 days of synthetic patient data (4 LINQ events, 5 workouts, 4 symptom entries, 7 sleep entries) to demonstrate the full feature set immediately.

Demo data can also be reloaded from **Settings → Reload Demo Data**.

---

## Sharing Reports

The Report screen uses React Native's built-in [`Share` API](https://reactnative.dev/docs/share) to produce a formatted plain-text summary that can be emailed, messaged, or AirDropped to a clinician.

## Clinical / Regulatory Note

This prototype is intended for **demonstration and patient engagement research** only. It:
- Does **not** interface with the LINQ device hardware or Medtronic's clinical networks
- Does **not** store or transmit data to any external server
- Is **not** FDA-cleared or CE-marked
- Should **not** be used as the basis for any clinical decision

Before any patient-facing deployment, a full regulatory review and clinical validation would be required.
